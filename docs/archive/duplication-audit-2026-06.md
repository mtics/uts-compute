## Executive summary

This is a single large duplication colony rooted in a scoping-and-enforcement failure: lib/shared.ts was created to de-duplicate exactly three data-movement modules (its own header says so) and only those three ever imported it, while the older job/access/quota/submit cluster kept private copies and access.ts independently grew its own exported copies of the SSH/timeout validators — becoming an accidental SECOND hub. The 75 findings dedup to ~48 consolidation units across security validators (SSH target/remote-path/realpath guards), the process-executor + timeout-bounds family, redaction/PII masking, evidence-file I/O, deterministic-JSON hashing, embedded Python helpers, and the index.ts/prompts.ts zod schema layer. The dominant pattern — every large module carries a ~40-line tail of copy-pasted assertSafe*/normalize*/redact*/default*Executor helpers — makes copy-paste the path of least resistance, so the fix must pair a broadened lib/ structure with a CI lint rule forbidding local re-definition of an already-exported name. Critically, several divergences are load-bearing and must be parameterized not flattened: per-module timeout bounds (transfer's 600s cap, access's 5s default), the supervisor-inner-hop SSH-hardening gap (a genuine security drift requiring a deliberate StrictHostKeyChecking policy), the contradictory runId regexes that block schema-layer tightening, approvals.canonicalJson's undefined-handling (swapping breaks issued approval ids), and the exact-vs-substring redaction modes (a naive merge causes audit-log under-redaction).

## Root cause

lib/shared.ts was introduced mid-development as a targeted fix for ONE cluster — its own header says 'Low-level helpers shared by the data-movement / state modules (artifacts, transfer, migrations)' — and only those exact 3 modules ever imported it. The older, larger job/access/quota/submit cluster predated that refactor and was never migrated. Worse, access.ts had independently grown its own copies of the SSH validators and timeout helpers and EXPORTS them, so jobs/quotas/ihpc-start/doctor wired up to access.ts as a de-facto SECONDARY hub instead of lib/shared.ts. This produced a three-way split (lib/shared.ts canonical home, access.ts accidental hub, submit.ts fully private copies) for the very same byte-identical functions. The systemic cause is therefore (a) a shared module scoped to a subset of modules by its own comment rather than the whole server, (b) no lint/CI rule forbidding local re-definition of a name that lib/shared.ts already exports, and (c) the dominant authoring pattern of appending private helpers to the bottom of each large module (every module has its ~40-line tail of assertSafe*/normalize*/redact*/default*Executor helpers), which makes copy-paste the path of least resistance. Three partial sinks already exist (lib/shared.ts, ids.ts, paths.ts) and prove the team accepts the pattern — the failure is reach and enforcement, not acceptance. Regression prevention: (1) broaden lib/shared.ts's charter comment to 'all modules' and split it into named sub-modules; (2) add an ESLint no-restricted-syntax / a custom check that fails CI when a module locally defines a function whose name is already exported from lib/; (3) make access.ts re-export from lib/shared.ts (never define) so the hub can't drift; (4) a one-time codemod migrating the bypassing cluster.

## Target lib/ structure

Promote lib/ from a single shared.ts (scoped to 3 modules) into a small set of charter-wide leaf modules, and elevate the two existing partial sinks (ids.ts, paths.ts) to authoritative status. Proposed layout (all leaf modules, no domain imports, to avoid cycles):

lib/shared.ts (keep, broaden charter to ALL modules): pure primitives — canonicalize, stableJson, safeTimestamp, safeTimestampOf, stripTrailingSlash, isInsideRemoteRoot, isHexDigest/assertHexDigest, sha256Hex, sha256File, boundedInteger, encodeSpec.

lib/ssh.ts (NEW): assertSafeSshTarget, sshTimeoutSeconds, the SSH-hardening flag constants, sshExecArgs(host,timeoutMs,remoteArgv,{tty}), sshTransportFlags, sshInnerHopFlags, isSafeRemotePath/assertSafeRemotePath. access.ts re-exports from here (never defines) so the secondary hub can't drift; submit/jobs/quotas/ihpc-start/artifacts/transfer import from here.

lib/process.ts (NEW): runProcess(program,args,timeoutMs,stdin?), the Executor type, and the relocated CommandResult interface. This is the transport primitive doctor/quotas should depend on instead of access.ts.

lib/redact.ts (NEW): redactWithTokens, summarizeRemoteFailure({label,timeoutMsg}), maskCommandArgs(args,replacements,{mode}), maskProjectRelative, maskArtifactCachePath. Delegates host/path masking to config.maskUserRootPath.

lib/evidence.ts (NEW): writeEvidenceJson(dir,fileName,record,label), runRecordRoot(baseDir,runId,label). Pairs with paths.resolveRecordPath.

lib/remote-python.ts (NEW): PREAMBLE_SIMPLE, PREAMBLE_CODED, INSIDE_REALPATH, SHA256_FILE snippet constants + PY_DECODE_PREAMBLE; the 8 helper bodies composed from these named pieces, addressed by name.

lib/pbs.ts (NEW): one walltime parser (number|undefined, optional days field), isQstatJobRow, and (if reconciled) a parameterized qstat kv tokenizer.

ids.ts (existing, elevate): SAFE_RUN_ID_PATTERN + assertSafeRunId + assertSafeApprovalId (+ SAFE_APPROVAL_ID_PATTERN) + assertSafeBoundedId(value,label,{maxLen,allowColon}); the single home for ALL id grammars.

paths.ts (existing, elevate): add RUNTIME_DIRS registry, resolveRecordPath, assertRealPathInsideRealRoot alongside the existing assertInsideRuntime/assertRealPathInside.

schemas.ts (NEW): all reusable zod field factories and constants (timeoutMsField, id/hash constants, PLATFORM_ENUM/HINT, byte-cap fields, ARTIFACT_IDS_FIELD/SWEEP_PARAMETERS_FIELD/JSON_OBJECT) consumed by index.ts and prompts.ts.

Net: ~8-9 leaf modules under lib/ plus elevated ids.ts/paths.ts/schemas.ts, with access.ts demoted from accidental hub to re-exporter, eliminating the three-way split.

## Deliberately deferred / rejected (do NOT naively merge)

Several findings were rejected, downgraded, or deliberately kept separate: (1) approvals.canonicalJson swap to stableJson — REJECTED as drop-in: it is self-consistent (never compared cross-side), the undefined-divergence is unreachable through real JSON-parse entry points, and swapping would change every existing scope_hash/approvalId and break idempotent re-request matching (approvals.ts:193). Keep separate pending a deliberate hash-version bump. (2) Promoting SAFE_RUN_ID_PATTERN to the index.ts runId schema — REJECTED: the handler-layer runId validators are mutually contradictory (lowercase {3,128} vs mixed-case {1,160}); tightening the schema would reject currently-accepted/persisted ids. The schema-layer DRY (bare z.string() constants) is safe; the regex promotion is a separate reconciliation task. (3) clampFactor (retry.ts:195) — kept SEPARATE from boundedInteger: Number.isFinite (allows fractional 1.5) + undefined-passthrough are load-bearing. (4) normalizeMaxTotalBytes — kept as a thin undefined-passthrough wrapper, NOT folded raw into boundedInteger (would impose a hardcoded cap where absence means unlimited). (5) assertSafeOutputToken (artifacts.ts:1827) — deliberately NOT folded into isSafeRemotePath (intentionally omits the absolute-path requirement). (6) sshOnNode inner hop + supervisor inner hop — NOT mechanically merged; require a conscious StrictHostKeyChecking policy decision. (7) docs.ts resolveDocPath — NOT folded into resolveRecordPath (realpath-first, different root). (8) WebUI 'HTML evidence rows' third copy of the requested-resources join — does not exist (false claim in finding). (9) The original 'lib/shared imported by only 3 of 40' finding's claim that all 7 exports are reimplemented and that the whole job/access/quota cluster redefines every export — MISCHARACTERIZED: only 3 of the exports (assertSafeSshTarget, sshTimeoutSeconds, canonicalize/stableJson) are duplicated outside the importers; stripTrailingSlash/isInsideRemoteRoot/safeTimestamp are largely correctly adopted; access.ts is a secondary hub (re-exports) not a wholesale re-implementer.
